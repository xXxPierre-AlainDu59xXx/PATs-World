
Kurzbeschreibung für den Aufruf eines TAM Web-Services der - basierend auf einer Object-GUID -
Properties aus dem AD ausliest und an den Aufrufer im Response zurückliefert :

parameters
  <objectguid></objectguid> 
  <domainfqdn></domainfqdn> 
  <cfgid></cfgid> 

Beispielhafter vollständiger Request :

<cmrequest cmd="" instance="" session="">
  <fingerprint>101</fingerprint> 
  <objectguid>a69c0336-9f74-4ebd-b2fb-c973abb5c6ef</objectguid> 
  <domainfqdn>central.econet.de</domainfqdn> 
  <cfgid>urm.test_1</cfgid> 
</cmrequest>


Die Rückgabe kann mittels XSLT (bzw. COM PlugIn) frei gewählt werden, als Default wird folgende
Struktur generiert, wobei das Element "anynode" wiederum in der Konfiguration des TAM Services
beliebig gewählt werden kann.

Beispiel :

<cmresponse instance="" session="" job="" /> 
  <anynode>
    <anynode>
      <properties>
        <property mode="1" multivalued="0" typename="String">
          <name>Guid</name> 
          <value>
          <![CDATA[ 36039ca6749fbd4eb2fbc973abb5c6ef
          ]]> 
          </value>
        </property>
        <property mode="1" multivalued="-1" typename="Variant()">
          <name>memberOf</name> 
          <value>
          <![CDATA[ CN=group1,OU=mygroups,DC=central,DC=econet,DC=de
          ]]> 
          </value>
          <value>
          <value>
          <![CDATA[ CN=group2,OU=mygroups,DC=central,DC=econet,DC=de
          ]]> 
          </value>
          <value>
          <![CDATA[ CN=group3,OU=mygroups,DC=central,DC=econet,DC=de
          ]]> 
          </value>
        </property>
      </properties>
    </anynode>
  </anynode>
</cmdata>

Eine entsprechende Konfiguration (Teil von "wse_config.xml") des TAM services könnte folgendermassen aussehen :

<app_adobj_details xmlid="">
  <cfg id="urm.test_1" xslid="" progid="" method="">
    <template>
      <anynode>
        <anynode>
          <properties>
            <property mode="1">
              <name>Guid</name> 
            </property>
            <property mode="1">
              <name>memberOf</name> 
            </property>
          </anynode>
        </anynode>
      </template>
    <dcfqdn>*</dcfqdn> 
  </cfg>
</app_adobj_details>
